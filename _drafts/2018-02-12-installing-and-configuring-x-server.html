Doing a full xorg install will clobber my machine with loads of perfunctory packages and dependencies so I did a minimal install of xorg.

Xbps-install xorg-minimal xorg-fonts

I then needed to install the right graphics driver.
I ran dmesg ! Grep amd
Copy and past from other post


xbps-install xf86-video-amdgpu

Finally to get a base xserver configuration going I installed a basic terminal emulator, a simple window manager, and a clock application

Xbps-install xterm xclock twm

To kick off x, I simply run
Startx

Now that x has been installed, it needs to be configured.

i3-gaps has been previously installed, along with i3status. I need this to launch automatically when I run startx.
Xorg-minimal contains a package Called xinit. See man xinit for more details.
Basically xinit starts the x windows system. Users configuration parameters are contained In a file called .xinitrc, in the users home directory.
In this file type in Exec i3.





Previously I had a system configuration command in my i3 config file namely:

exec xrdb /home/eamoc/.XResources

 where it would have been more appropriate to run the command with at the beginning of the X Server session, before the Window Manager starts. The way to do this is to create the .xprofile file in the home directory.
